Generative AI — the foundation

What is Generative AI?

Generative AI is a type of Artificial Intelligence that can create new content — text, images, audio, video, code — instead of just analyzing existing data.

Think of it like this: traditional AI is a detective who looks at data and decides "this email is spam." Generative AI is an artist or writer — it produces something new.

Simple example: You type "Write a poem about the moon." Generative AI replies: "Silver glow upon the sea, the moon dances silently…" It didn't copy a poem from anywhere. It generated a new one based on patterns it learned during training.

How did it get so powerful?

Modern generative AI is built on Deep Learning and a special architecture called the Transformer (introduced by Google in 2017 in the paper "Attention Is All You Need"). The Transformer is the engine behind models like GPT, Gemini, Claude, and Llama.

Types of Generative AI

TypeWhat It CreatesExample Tools
Text generationArticles, code, answersChatGPT, Claude, Gemini
Image generationPictures, artDALL·E, Midjourney, Stable Diffusion
Audio generationMusic, speechElevenLabs, Suno
Video generationShort videosSora, Runway
Code generationPrograms, scriptsGitHub Copilot, Cursor
LangChain is the toolkit that helps developers build applications using these generative AI models — especially LLMs.
02

Large Language Models

What is an LLM?

LLM = Large Language Model. It's a giant neural network trained on huge amounts of text from the internet, books, articles, and code. After training, it can answer questions, write essays, translate languages, write code, summarize documents, and have conversations.

Why "Large"?

Because these models have billions of parameters (parameters are like little knobs the model adjusts during training).

  • GPT-3: 175 billion parameters
  • Llama 3: up to 405 billion parameters
  • GPT-4: estimated 1+ trillion parameters

More parameters = generally smarter, but also more expensive to run.

How does an LLM actually work?

Imagine you read every book in the world. After reading, if I give you the sentence "The capital of France is ___", you will instantly say "Paris" because you've seen it thousands of times. An LLM works similarly — it predicts the next word based on patterns. It does this one word at a time, over and over, building up full responses.

LLMs don't "understand" like humans. They predict the most likely next token (word piece) based on patterns. But because they've seen so much text, the result feels intelligent.

Popular LLMs (Quick List)

  • GPT-4 / GPT-4o — by OpenAI
  • Claude — by Anthropic
  • Gemini — by Google
  • Llama 3 — by Meta (open source)
  • Mistral / Mixtral — by Mistral AI (open source)
  • Gemma — by Google (open source)
03

Paid LLMs vs Open Source LLMs

This is a very important topic. There are two main types of LLMs you'll work with.

Paid LLMs (Closed Source, API-based)

You don't get the model itself. You pay a company to send requests to their model over the internet (an API).

Examples: OpenAI → GPT-4, GPT-4o · Anthropic → Claude · Google → Gemini Pro.

How it works

  1. You sign up and get an API key (like a password).
  2. In your code, you send a question to the API.
  3. They send back an answer.
  4. You're charged per token (a token ≈ ¾ of a word).
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o", api_key="your-secret-key")
response = llm.invoke("What is LangChain?")
print(response.content)

Pros

  • Very high quality (state-of-the-art)
  • No need for powerful hardware
  • Easy to use
  • Always updated

Cons

  • Costs money (can get expensive)
  • Your data leaves your computer
  • Internet required
  • Vendor lock-in

Open Source LLMs

These are LLMs whose weights (the model itself) are publicly available. You can download them and run them on your own computer or server.

Examples: Llama 3 (Meta), Mistral 7B / Mixtral (Mistral AI), Gemma (Google), Phi-3 (Microsoft), Qwen (Alibaba), DeepSeek.

How to run them

  • Ollama — the easiest way (a tool to run open-source LLMs locally on your laptop)
  • Hugging Face Transformers — Python library to load models
  • Groq — runs open-source models on super-fast cloud hardware (often free tier)
# Using Ollama (runs locally)
from langchain_community.llms import Ollama

llm = Ollama(model="llama3")
response = llm.invoke("Explain gravity in one line.")
print(response)
# Using Groq (free + super fast)
from langchain_groq import ChatGroq

llm = ChatGroq(model="llama3-70b-8192", api_key="your-groq-key")
response = llm.invoke("Hello!")
print(response.content)

Multimodal Models

What does "multimodal" mean?

Modal = a type of input (text, image, audio, video). Multimodal = a model that can handle multiple types of input or output, not just text.

You upload a picture of a fridge and ask: "What can I cook with what's inside?" A multimodal model like GPT-4o or Gemini can look at the image, identify ingredients (eggs, tomatoes, cheese), and suggest: "You can make a tomato-cheese omelet!"

Popular multimodal models

  • GPT-4o (OpenAI) — text + image + audio
  • Gemini 1.5 / 2.0 (Google) — text + image + video + audio
  • Claude 3+ (Anthropic) — text + image
  • LLaVA (open source) — text + image

Multimodal in LangChain

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

llm = ChatOpenAI(model="gpt-4o")

message = HumanMessage(
    content=[
        {"type": "text", "text": "What's in this image?"},
        {"type": "image_url",
         "image_url": {"url": "https://example.com/cat.jpg"}},
    ]
)

response = llm.invoke([message])
print(response.content)

Messages & Prompts

What is a prompt?

A prompt is the instruction or question you send to the LLM.

  • Bad prompt: "Tell me stuff."
  • Good prompt: "You are a science teacher. Explain photosynthesis to a 10-year-old in 3 sentences."

The better the prompt, the better the answer. This is called prompt engineering.

Three types of messages

SystemMessage

Sets the role / behavior of the AI. Sent at the start. "You are a polite assistant who answers in formal English."

HumanMessage

The user's question or input. "What is the speed of light?"

AIMessage

The LLM's reply. "The speed of light is approximately 299,792 km/s."

from langchain_core.messages import SystemMessage, HumanMessage
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

messages = [
    SystemMessage(content="You are a helpful math tutor."),
    HumanMessage(content="What is 25 times 4?"),
]

response = llm.invoke(messages)
print(response.content)  # "25 times 4 is 100."

PromptTemplate — reusable prompts

Instead of hardcoding prompts, use a template with placeholders.

from langchain_core.prompts import PromptTemplate

template = PromptTemplate.from_template(
    "Translate this sentence to {language}: {sentence}"
)

prompt = template.format(language="French", sentence="Good morning")
print(prompt)
# Translate this sentence to French: Good morning
A PromptTemplate is like a Mad Libs game. You have a sentence with blanks, and you fill them in differently each time.

Chains — the heart of LangChain

What is a Chain?

A Chain is a sequence of steps that connect components together. The output of one step becomes the input of the next.

The word "LangChain" literally comes from this — chaining language model calls together.

Simplest chain: Prompt → LLM → Output

You take a prompt template, an LLM, and an output parser. Link them with the | (pipe) operator — this is called LCEL (LangChain Expression Language).

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a translator."),
    ("human", "Translate '{text}' to {language}.")
])

parser = StrOutputParser()

# Chain them together with pipes
chain = prompt | llm | parser

result = chain.invoke({"text": "Hello, how are you?",
                       "language": "Spanish"})
print(result)  # "Hola, ¿cómo estás?"
A chain is like an assembly line in a factory: Station 1 — raw materials enter (prompt). Station 2 — machine works on it (LLM). Station 3 — packaging (parser). Final product comes out the end.

Types of chains

  1. Simple Chain (LCEL) — prompt → LLM → output (most common today)
  2. Sequential Chain — multiple LLM calls one after another
  3. Router Chain — decides which sub-chain to use based on input
  4. Stuff / Map-Reduce / Refine Chains — used for summarization

Example: sequential chain

Generate a blog title from a topic, then generate a blog post from that title:

title_prompt = ChatPromptTemplate.from_template(
    "Give me a catchy blog title about {topic}."
)
post_prompt = ChatPromptTemplate.from_template(
    "Write a 100-word blog post titled: {title}"
)

title_chain = title_prompt | llm | StrOutputParser()
post_chain = post_prompt | llm | StrOutputParser()

full_chain = {"title": title_chain} | post_chain
result = full_chain.invoke({"topic": "space exploration"})
print(result)

Memory & Conversation History

The problem

LLMs are stateless — they don't remember the previous message. Every call is fresh.

Without memory:
You: My name is Rahul.
AI: Hello Rahul!
You: What's my name?
AI: I don't know your name. ❌

The solution: memory

LangChain provides memory that stores the conversation and feeds it back to the LLM each time.

Types of memory

TypeWhat it does
ConversationBufferMemoryStores all messages (can get long)
ConversationBufferWindowMemoryStores only last N messages
ConversationSummaryMemorySummarizes old chats to save tokens
ConversationSummaryBufferMemoryMix: summary of old + recent messages

Modern approach: RunnableWithMessageHistory

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_community.chat_message_histories import ChatMessageHistory
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a friendly assistant."),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{input}")
])

chain = prompt | llm
store = {}

def get_session_history(session_id):
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

chatbot = RunnableWithMessageHistory(
    chain, get_session_history,
    input_messages_key="input",
    history_messages_key="history",
)

config = {"configurable": {"session_id": "user1"}}

chatbot.invoke({"input": "Hi, I'm Rahul."}, config=config)
chatbot.invoke({"input": "What's my name?"}, config=config)
# "Your name is Rahul." ✓
Memory is like a notebook a waiter carries. Without it, the waiter forgets your order every time you speak. With it, they remember everything for your whole meal.

from langchain_community.document_loaders import PyPDFLoader

loader = PyPDFLoader("mybook.pdf")

print(documents[0].page_content[:200])

Text Splitters

Why split? LLMs have a context window limit (e.g., 8,000 tokens). A 300-page book won't fit. So we split it into smaller chunks.

The most common splitter is RecursiveCharacterTextSplitter — it tries to split on paragraphs first, then sentences, then words.

from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,     # each chunk = 500 characters
    chunk_overlap=50    # overlap (prevents cutting mid-sentence)
)

chunks = splitter.split_documents(documents)
print(f"Total chunks: {len(chunks)}")
Imagine studying a 500-page textbook. You can't memorize it all at once. So you break it into chapters and sections — that's text splitting.

Embeddings — text into numbers

What are embeddings?

Computers don't understand words — they understand numbers. An embedding converts a piece of text into a vector (a list of numbers, usually 384, 768, or 1536 numbers long).

"Dog" → [0.21, -0.87, 0.45, 0.12, …]
"Puppy" → [0.20, -0.85, 0.46, 0.13, …]   very similar to dog
"Car" → [-0.55, 0.32, -0.91, 0.04, …]   very different

Why useful?

Because similar meanings produce similar vectors, we can do semantic search — find text that means the same thing, even if the words differ. "Automobile" and "car" have similar embeddings, so a search for "car" can find "automobile."

Popular embedding models

ModelProviderType
text-embedding-3-smallOpenAIPaid
text-embedding-3-largeOpenAIPaid
all-MiniLM-L6-v2HuggingFaceOpen source, fast
BGEBeijing AIOpen source, strong
Cohere EmbeddingsCoherePaid
 
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

vector = embeddings.embed_query("What is AI?")
print(len(vector))   # 1536 numbers
print(vector[:5])    # first 5: [0.024, -0.018, ...]
# Open-source alternative
from langchain_huggingface import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")
vector = embeddings.embed_query("What is AI?")
Think of embeddings as GPS coordinates for ideas. Two cities physically close (Mumbai and Pune) have similar coordinates. Two words with close meaning (happy and joyful) have similar embeddings.

Vector Databases

What is a Vector Database?

A vector database stores all those embedding vectors and lets you search them lightning-fast.

When you ask a question: your question is converted to a vector, the vector DB finds the most similar vectors stored in it, and returns the matching text chunks.

Why not a regular database?

A normal database (like MySQL) searches by exact match: WHERE name = 'Rahul'. A vector DB searches by meaning using math (cosine similarity, dot product).

Popular vector databases

Vector DBTypeNotes
FAISSLocal libraryMade by Meta, very fast, free
ChromaLocal / lightweightEasy to use, great for prototypes
PineconeCloud (paid)Production-grade, scalable
WeaviateOpen source / cloudBuilt-in hybrid search
QdrantOpen sourceFast, written in Rust
MilvusOpen sourceIndustry-scale
AstraDBCloudBy DataStax

Example: storing & searching with FAISS

from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

texts = [
    "The sun rises in the east.",
    "Cats are mammals.",
    "Python is a programming language.",
    "The Eiffel Tower is in Paris.",
]

vectorstore = FAISS.from_texts(texts, embeddings)

query = "Where is the Eiffel Tower?"
results = vectorstore.similarity_search(query, k=1)
print(results[0].page_content)
# "The Eiffel Tower is in Paris."
A vector DB is like a librarian with a perfect memory of meaning. You don't have to know the exact title of a book — just describe what it's about, and they'll bring you the right one.

Retrievers & RAG

This is the most important concept after LLMs themselves. Pay close attention.

The problem with LLMs alone

LLMs don't know your private data (company docs, your notes), have a knowledge cutoff (don't know recent events), and can hallucinate — make up facts confidently.

The solution: RAG

RAG = Retrieval Augmented Generation. The idea: take the user's question, retrieve relevant info from YOUR documents using a vector DB, augment the prompt with that info, and let the LLM generate the answer based on the real data.

What is a Retriever?

A Retriever is a LangChain object whose job is to fetch documents. The most common one wraps a vector store.

retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
docs = retriever.invoke("How does photosynthesis work?")

Complete RAG example (Q&A over a PDF)

from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain.chains import create_retrieval_chain

# 1. Load PDF
loader = PyPDFLoader("biology.pdf")
docs = loader.load()

# 2. Split into chunks
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)

# 3. Embed and store
vectorstore = FAISS.from_documents(chunks, OpenAIEmbeddings())
retriever = vectorstore.as_retriever()

# 4. Build prompt
prompt = ChatPromptTemplate.from_template("""
Answer the question based ONLY on the following context:
{context}

Question: {input}
""")

# 5. Build chain
llm = ChatOpenAI(model="gpt-4o")
document_chain = create_stuff_documents_chain(llm, prompt)
retrieval_chain = create_retrieval_chain(retriever, document_chain)

# 6. Ask!
response = retrieval_chain.invoke({"input": "What is mitosis?"})
print(response["answer"])

Types of Retrievers

RetrieverWhat it does
VectorStoreRetrieverStandard semantic search
MultiQueryRetrieverRephrases query multiple ways for better recall
ContextualCompressionRetrieverTrims retrieved docs to keep only relevant parts
EnsembleRetrieverCombines retrievers (e.g., keyword + semantic)
ParentDocumentRetrieverRetrieves small chunks but returns larger parent docs
RAG is like an open-book exam: a student (LLM) may not know everything, but if they can quickly look up the right textbook page (retriever), they can give a perfect answer using that info.

Text Summarizer

A text summarizer takes a long document and gives you a short version. LangChain provides built-in strategies for this since long documents won't fit in an LLM's context window.

Three main strategies

1. Stuff — throw everything at the LLM

Works only for short documents that fit in one prompt. Fast and accurate when feasible.

2. Map-Reduce

Map step: summarize each chunk separately. Reduce step: combine the chunk summaries into one final summary. Works for long documents.

3. Refine

Summarize chunk 1. Then look at chunk 2 + previous summary → improve summary. Continue for each chunk. Slow but quality is often best.

Example: Map-Reduce summarization

from langchain.chains.summarize import load_summarize_chain
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_openai import ChatOpenAI

loader = PyPDFLoader("long_report.pdf")
docs = loader.load()

splitter = RecursiveCharacterTextSplitter(chunk_size=2000, chunk_overlap=200)
chunks = splitter.split_documents(docs)

llm = ChatOpenAI(model="gpt-4o")
chain = load_summarize_chain(llm, chain_type="map_reduce")
summary = chain.invoke({"input_documents": chunks})

print(summary["output_text"])
Summarizing all 7 Harry Potter books: Stuff = read all 7 and write one summary (impossible in one sitting). Map-Reduce = summarize each book separately, then combine the 7 summaries. Refine = read book 1 and summarize, then update the summary after book 2, etc.

Use cases

  • Summarizing news articles
  • Meeting transcript → action items
  • Research paper → key findings
  • Long emails → quick brief
  • YouTube videos → bullet points

Chatbots

A chatbot is one of the most common LangChain projects. It combines an LLM, prompt template, message history (memory), optionally RAG for domain knowledge, and optionally tools.

Types of chatbots

  1. Simple Q&A chatbot — answers from LLM's own knowledge
  2. Document chatbot (RAG) — answers based on your PDFs/files
  3. Customer support chatbot — answers from FAQs + can take actions
  4. Agentic chatbot — can use tools (search, calculator, APIs)

Basic chatbot code with memory

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory

from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Reply concisely."),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{input}")
])

chain = prompt | llm
store = {}

def get_session_history(session_id):
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

chatbot = RunnableWithMessageHistory(
    chain, get_session_history,
    input_messages_key="input",
    history_messages_key="history"
)

config = {"configurable": {"session_id": "abc123"}}
print(chatbot.invoke({"input": "Hi, I'm learning AI."}, config=config).content)
print(chatbot.invoke({"input": "Recommend a beginner book."}, config=config).content)

Pro tip: streaming

To get the chatbot to "type out" answers word by word (like ChatGPT):

for chunk in chatbot.stream({"input": "Tell me a story"}, config=config):
    print(chunk.content, end="", flush=True)

Chat with SQL Database

The goal

You have a database (e.g., a company's sales DB). You want to ask questions in plain English, and the system should: convert English → SQL query, run the query on the DB, return the answer in English.

This is called Text-to-SQL or an SQL Agent.

A non-technical manager can ask: "How many products did we sell last month in Mumbai?" — and get an answer immediately, without knowing SQL.

Step-by-step example

from langchain_community.utilities import SQLDatabase
from langchain_community.agent_toolkits import create_sql_agent
from langchain_openai import ChatOpenAI

# 1. Connect to the database
db = SQLDatabase.from_uri("sqlite:///sales.db")
# Or: "mysql+pymysql://user:pass@localhost/dbname"
# Or: "postgresql+psycopg2://user:pass@localhost/dbname"

# 2. Initialize LLM
llm = ChatOpenAI(model="gpt-4o", temperature=0)

# 3. Create SQL agent
agent_executor = create_sql_agent(llm, db=db,
                                   agent_type="openai-tools",
                                   verbose=True)

# 4. Ask in English!
response = agent_executor.invoke(
    {"input": "How many orders were placed in January?"}
)
print(response["output"])

What happens internally

  1. Agent inspects the database structure (tables, columns).
  2. Builds a prompt: "Given these tables, write SQL for: 'How many orders in Jan?'"
  3. LLM produces: SELECT COUNT(*) FROM orders WHERE month='January';
  4. Agent runs the query against your database.
  5. Agent reads the result and converts it to plain English: "There were 245 orders placed in January."
For real applications, restrict the database to read-only access so the agent can't accidentally delete or modify data.
Imagine hiring a translator who knows SQL. You speak English, they speak SQL to the database, and bring back the answer in English.

Agents & Tools

What is an Agent?

Up to now, every chain has been fixed — we predefined the steps. An Agent is smarter. It decides on its own which tools to use to solve a problem.

The core idea

You give an agent a goal (the user's question) and a set of tools (search, calculator, weather API, database, etc.). The agent plans which tool to use, uses it, looks at the result, and decides if it's done or needs another tool. It repeats until it has an answer.

Question: "What's the temperature in Tokyo right now multiplied by 7?"

Agent thinks: "I don't know the current temp. Let me use the weather tool." → 28°C.
"Now I need 28 × 7. Let me use the calculator tool." → 196.
"I'm done. Final answer: 196."

Agent with Search + Wikipedia

from langchain_community.tools import WikipediaQueryRun, DuckDuckGoSearchRun
from langchain_community.utilities import WikipediaAPIWrapper
from langchain.agents import create_react_agent, AgentExecutor
from langchain import hub
from langchain_openai import ChatOpenAI

# Tools
wiki = WikipediaQueryRun(api_wrapper=WikipediaAPIWrapper())
search = DuckDuckGoSearchRun()
tools = [wiki, search]

# LLM + prompt
llm = ChatOpenAI(model="gpt-4o")
prompt = hub.pull("hwchase17/react")

# Create agent
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

result = executor.invoke({"input": "Who won the 2024 Nobel Prize in Physics?"})
print(result["output"])

What is ReAct?

ReAct = Reason + Act. The agent alternates between:

  • Reason — Think: "What should I do next?"
  • Act — Take action: call a tool
  • Observation — Look at the result
  • Repeat.

from langchain_core.tools import tool

@tool

    """Returns the length of a word."""

Streamlit — the user interface

What is Streamlit?

Your LangChain code runs in the terminal. To turn it into a real web app (with a text box, buttons, chat interface), you use Streamlit — a Python library that turns scripts into beautiful web apps with very little code.

Why it's loved

  • Pure Python (no HTML/CSS/JavaScript needed)
  • Reload-on-save (instant feedback)
  • Free and open source
  • Easy to deploy (Streamlit Cloud, Hugging Face Spaces)
pip install streamlit
streamlit run app.py

import streamlit as st
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage

st.title("🤖 My LangChain Chatbot")

if "messages" not in st.session_state:
    st.session_state.messages = [SystemMessage(
        content="You are a helpful assistant.")]

# Display past messages
for msg in st.session_state.messages[1:]:
    role = "user" if isinstance(msg, HumanMessage) else "assistant"
    with st.chat_message(role):
        st.write(msg.content)

user_input = st.chat_input("Type your message...")

if user_input:
    st.session_state.messages.append(HumanMessage(content=user_input))
    with st.chat_message("user"):
        st.write(user_input)

    llm = ChatOpenAI(model="gpt-4o")
    response = llm.invoke(st.session_state.messages)

    st.session_state.messages.append(AIMessage(content=response.content))
    with st.chat_message("assistant"):
        st.write(response.content)

PDF Q&A app in Streamlit

import streamlit as st
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain.chains import create_retrieval_chain
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate
import tempfile

st.title("📄 Chat with your PDF")

uploaded_file = st.file_uploader("Upload a PDF", type="pdf")

if uploaded_file:
    with tempfile.NamedTemporaryFile(delete=False, suffix=".pdf") as tmp:
        tmp.write(uploaded_file.read())
        path = tmp.name

    loader = PyPDFLoader(path)
    docs = loader.load()
    splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
    chunks = splitter.split_documents(docs)

    vectorstore = FAISS.from_documents(chunks, OpenAIEmbeddings())
    retriever = vectorstore.as_retriever()

    llm = ChatOpenAI(model="gpt-4o")
    prompt = ChatPromptTemplate.from_template(
        "Answer based on context:\n{context}\n\nQuestion: {input}"
    )
    doc_chain = create_stuff_documents_chain(llm, prompt)
    retrieval_chain = create_retrieval_chain(retriever, doc_chain)

    question = st.text_input("Ask something about the PDF:")
    if question:
        with st.spinner("Thinking..."):
            result = retrieval_chain.invoke({"input": question})
        st.write(result["answer"])

Putting It All Together

The complete project flow

1.  DEFINE THE GOAL
    → "I want a chatbot that answers from my company's PDFs."

2.  PICK YOUR LLM
    → Paid (OpenAI/Anthropic) OR open source (Llama via Groq/Ollama)

3.  LOAD YOUR DATA
    → PyPDFLoader, WebBaseLoader, CSVLoader, etc.

4.  SPLIT DATA INTO CHUNKS
    → RecursiveCharacterTextSplitter

5.  EMBED + STORE
    → OpenAIEmbeddings or HuggingFaceEmbeddings
    → FAISS / Chroma / Pinecone

6.  CREATE RETRIEVER
    → vectorstore.as_retriever()

7.  BUILD PROMPT TEMPLATE
    → ChatPromptTemplate with system + history + user input

8.  ADD MEMORY (if conversational)
    → RunnableWithMessageHistory

9.  BUILD CHAIN OR AGENT
    → Chain (fixed flow) OR Agent (decides tools dynamically)

10. WRAP IN STREAMLIT UI
    → st.chat_input(), st.chat_message(), st.file_uploader()

11. DEPLOY
    → Streamlit Cloud, Hugging Face Spaces, Render, Railway
